### Day 4：推理与效果对比（形成闭环）

**今日目标**

- [ ] 验证"训练是否真的带来可观察变化"

**今日任务**

- [ ] 准备固定测试问题集（建议 10～20 条医学问题）
- [ ] 用训练前基础模型推理，保存输出
- [ ] 用训练后 checkpoint 推理，保存输出
- [ ] 按维度对比：准确性、完整性、稳定性、幻觉情况
- [ ] 小范围调整解码参数（temperature / top_p / max_new_tokens）

**训练前后推理对比要点**

| 维度       | 训练前             | 训练后             |
|------------|--------------------|--------------------|
| 医学术语   | 不稳定             | 应更准确           |
| 回答格式   | 随机               | 应符合模板         |
| 拒答率     | 高或随机           | 应对齐训练数据策略 |
| 幻觉程度   | 高                 | 应有所降低         |

**今日产出（建议文件）**

## 常见问题排查清单（速查）

- [ ] **OOM（显存不足）**：降 batch_size、降 max_length、开梯度累积、检查是否误开大模型参数
- [ ] **loss 不下降**：查数据质量、查模板拼接、查 label mask、查学习率是否过大 / 过小
- [ ] **推理效果差**：先看训练数据分布，再看推理参数，最后看训练轮次是否不足
- [ ] **训练能跑但无提升**：做训练前后同题对比，确认是否真的学到目标格式与知识
- [ ] **复现困难**：固定随机种子、记录依赖版本、保存完整命令与数据版本

> 建议：每次实验只改一个关键变量，避免结论混淆。
